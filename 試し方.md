# Swimm を試す手順

## 概要

Swimm を使って、コードに紐づいたドキュメント（`.sw.md`）を作成し、
コード変更時に Auto-sync でドキュメントの乖離を自動検知できるか試す。

## 前提

- GitHubアカウントがあること
- GitHubにリポジトリを作成できること
- VS Code がインストールされていること（IDE連携で試す場合）

## ステップ1: GitHubにテスト用リポジトリを作成してpush

```bash
# このフォルダに移動
cd swimm-trial

# Gitリポジトリを初期化
git init
git add .
git commit -m "初期コミット: タスク管理APIのコードと設計書"

# GitHubに空のリポジトリを作成（ブラウザで作るか、gh CLIで作る）
gh repo create swimm-trial --public --source=. --push
# ※ プライベートリポジトリでもFreeプランで使える
```

**⚠️ セキュリティ注意:**
- 会社のリポジトリではなく、必ず新しいテスト用リポジトリを使うこと
- GitHub Appインストール時は「Only select repositories」を選び、テスト用リポジトリだけを指定すること

## ステップ2: Swimmアカウントを作成

1. https://app.swimm.io/ にアクセス
2. 「Sign up with GitHub」でGitHubアカウントを使ってサインアップ
3. ワークスペースを作成（名前は適当でOK、例: `test-workspace`）

**料金について:**
- Freeプラン: 5ユーザーまで無料（期間制限なし）
- クレジットカード登録不要

## ステップ3: リポジトリをSwimmに接続

1. Swimmのダッシュボードで「Add Repository」をクリック
2. GitHubを選択
3. Swimm GitHub Appのインストール画面が表示される
4. **「Only select repositories」を選択し、`swimm-trial` だけを指定**
5. インストールを完了すると、Swimmのダッシュボードにリポジトリが表示される

※ 参考: [Connect GitHub Cloud to Swimm（公式）](https://docs.swimm.io/getting-started-guide/connect-github-cloud-to-swimm/)

## ステップ4: Swimmドキュメントを作成する

### 方法A: Web Appで作成する場合

1. Swimmダッシュボードで `swimm-trial` リポジトリを開く
2. 「Create Doc」→「Blank doc」を選択
3. タイトルを入力（例: `タスク管理機能 内部設計書`）
4. 本文を書く。以下のSlashコマンドが使える:
   - `/Code snippet` — コードのスニペットを埋め込む（コードと連動する）
   - `/Smart Token` — 関数名や変数名をコードにリンクする
   - `/Path` — ファイルパスをリンクする
5. 例えば `TaskService.java` の `createTask` メソッドをスニペットとして埋め込む
6. 右上の「Commit」ボタンでリポジトリにコミットする
   - 直接 `main` にpushするか、新しいブランチを作ってPRにできる

### 方法B: VS Code IDE拡張で作成する場合

1. VS Codeの拡張機能で「Swimm」を検索してインストール
2. `swimm-trial` リポジトリをVS Codeで開く
3. Swimmパネルから「Create Doc」をクリック
4. `.swm/` フォルダにMarkdownファイルが作成される
5. `/Code snippet` コマンドでコードスニペットを埋め込む
6. 保存後、gitでコミット＆pushする

### ドキュメントの作り方のコツ

- `docs/design.md` の内容を参考にしつつ、Swimmのスニペット機能でコードを埋め込む
- 例: 「ステータス遷移ルール」の説明に `TaskService.java` の `isValidTransition` メソッドをスニペットとして追加
- Swimmのドキュメントは `.swm/` フォルダに `.sw.md` 形式で保存される

## ステップ5: コードを変更してPRを作成

設計書と連動しているコードを変更し、Auto-syncが乖離を検知するか試す。

### 変更例: 「優先度」フィールドを追加する

```bash
# 新しいブランチを作成
git checkout -b feature/add-priority
```

以下の変更を行う:

**Task.java に `priority` フィールドを追加:**
```java
private Priority priority;  // 追加

public Priority getPriority() { return priority; }  // 追加
public void setPriority(Priority priority) {         // 追加
    this.priority = priority;
    this.updatedAt = LocalDateTime.now();
}
```

**Priority.java を新規作成:**
```java
public enum Priority {
    LOW,
    MEDIUM,
    HIGH
}
```

**TaskService.java の createTask を修正:**
```java
public Task createTask(String title, String description, Priority priority) {
    Task task = new Task();
    task.setTitle(title);
    task.setDescription(description);
    task.setStatus(TaskStatus.TODO);
    task.setPriority(priority);  // 追加
    return taskRepository.save(task);
}
```

### コミットしてPRを作成

```bash
git add .
git commit -m "feat: タスクに優先度(priority)を追加"
git push origin feature/add-priority

# PRを作成
gh pr create --title "feat: タスクに優先度を追加" --body "タスクにLOW/MEDIUM/HIGHの優先度フィールドを追加"
```

## ステップ6: Auto-syncの動作を確認

PRを作成すると、SwimmのGitHub Appが自動でチェックを実行する。

- PRのChecksタブに「Swimm」のチェック結果が表示される
- ドキュメントに埋め込んだコードスニペットが変更されている場合:
  - **Auto-sync可能な場合**: Swimmが自動でドキュメント内のスニペットを更新し、修正コミットをPRに追加する
  - **Auto-sync不可能な場合**: PRにコメントで「このドキュメントの更新が必要」と警告が出る
- PRのChecksが ✅ になれば、ドキュメントとコードが同期している状態

### 確認ポイント

- [ ] PRのChecksに「Swimm」が表示されているか
- [ ] スニペットとして埋め込んだコード部分に変更があった場合、Auto-syncが動いたか
- [ ] Auto-syncで更新されたドキュメントの内容は正しいか
- [ ] Auto-syncできなかった場合、どのような警告が出るか

## DeepDocsとの違い・比較ポイント

| 観点 | Swimm | DeepDocs |
|---|---|---|
| ドキュメント形式 | `.sw.md`（独自拡張Markdown） | 既存の `.md` をそのまま使う |
| ドキュメント作成 | 手動（Web/IDE）でコードスニペット埋め込み | コードスキャンから自動生成 |
| コード変更時 | Auto-sync: スニペット部分を自動更新 | 自動スキャンで差分検知→更新PR作成 |
| 無料枠 | 5ユーザーまで無期限 | 7日間トライアルのみ |
| IDE連携 | VS Code / JetBrains 拡張あり | なし |

## うまくいかなかった場合

- SwimmダッシュボードのRepository Settingsを確認する
- GitHub AppがインストールされていてPRへのアクセス権があるか確認する
- ドキュメントにコードスニペットが正しく埋め込まれているか確認する（スニペットがないとAuto-syncは動かない）
- VS Code拡張を使っている場合、Swimmにログインしているか確認する
